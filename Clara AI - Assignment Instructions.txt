-  A  clear  changelog

##  Required  Output  Format  
 
###  1)  Account  Memo  JSON  (per  account)  
Create  a  structured  JSON  with  at  least  these  fields:  
-  account_id  
-  company_name  
-  business_hours  (days,  start,  end,  timezone)  
-  office_address  (if  present)  
-  services_supported  (list)  
-  emergency_definition  (list  of  triggers)  
-  emergency_routing_rules  (who  to  call,  order,  fallback)  
-  non_emergency_routing_rules  
-  call_transfer_rules  (timeouts,  retries,  what  to  say  if  fails)  
-  integration_constraints  (example:  “never  create  sprinkler  jobs  in  ServiceTrade”)  
-  after_hours_flow_summary  
-  office_hours_flow_summary  
-  questions_or_unknowns  (only  if  truly  missing)  
-  notes  (short)  
 
###  2)  Retell  Agent  Draft  Spec  (per  account)  
Produce  a  JSON  or  YAML  that  includes:  

-  agent_name  
-  voice  style  (basic)  
-  system  prompt  (your  generated  prompt)  
-  key  variables  (timezone,  business  hours,  address,  emergency  routing)  
-  tool  invocation  placeholders  (do  not  mention  tools  to  caller)  
-  call  transfer  protocol  
-  fallback  protocol  if  transfer  fails  
-  version  (v1  for  demo,  v2  after  onboarding  update)  
 
###  3)  Versioning  and  Diff  
When  onboarding  updates  arrive:  
-  produce  v2  memo  JSON  
-  produce  v2  agent  spec  
-  produce  a  changelog  file,  example:  `changes.md`  or  `changes.json`  
  -  show  what  changed  and  why  
 
###  4)  Orchestrator  Workflow  Export  
Provide:  
-  n8n  workflow  JSON  export  OR  Make  scenario  blueprint/screens  
-  plus  a  short  setup  guide  so  we  can  run  it  
 

###  5)  README  
A  clean  README  that  explains:  
-  architecture  and  data  flow  
-  how  to  run  locally  
-  how  to  plug  in  the  dataset  files  
-  where  outputs  are  stored  
-  known  limitations  
-  what  you  would  improve  with  production  access  
##  Functional  Requirements  
###  Extraction  and  Structuring  
-  Your  system  must  extract  key  data  from  transcripts  reliably.  
-  It  must  avoid  hallucinating  unknowns.  
-  If  a  detail  is  missing,  it  should  be  left  blank  or  flagged  under  
`questions_or_unknowns`.
 
###  Prompt  Hygiene  
Generated  agent  prompt  must  include:  
-  Business  hours  flow:  greeting,  purpose,  collect  name  and  number,  route  or  transfer,  
fallback
 
if
 
transfer
 
fails,
 
confirm
 
next
 
steps,
 
“anything
 
else”,
 
close.
 
-  After  hours  flow:  greet,  purpose,  confirm  emergency,  if  emergency  collect  name,  
number,
 
address
 
immediately,
 
attempt
 
transfer,
 
fallback
 
if
 
transfer
 
fails,
 
assure
 
quick
 
followup,
 
“anything
 
else”,
 
close.
 
-  Must  not  ask  too  many  questions.  Only  collect  what  is  needed  for  routing  and  
dispatch.
 

-  Must  not  mention  “function  calls”  to  the  caller.  
-  Must  include  a  clear  call  transfer  protocol  and  transfer-fail  protocol.  
###  Automation  Behavior  
-  Pipeline  A  auto-generates  v1  assets  for  each  demo  call.  
-  Pipeline  B  takes  onboarding  input  and  updates  the  same  account_id  to  v2.  
-  The  workflow  should  be  repeatable  and  idempotent  (running  twice  should  not  
create
 
chaos).
 
##  Suggested  Architecture  (You  Can  Improve  This)  
1)  Ingest  file  (recording  or  transcript)  ->  normalize  ->  assign  account_id  
2)  Transcription  step  (optional  if  transcripts  provided)  
3)  LLM  extraction  step  ->  Account  Memo  JSON  
4)  Prompt  generator  step  ->  Retell  Agent  Draft  Spec  
5)  Store  outputs  ->  repo  or  DB  
6)  Create  task  tracker  item  ->  Asana  or  alternative  
7)  Onboarding  input  ->  extract  updates  ->  apply  patch  ->  regenerate  agent  spec  v2  
->
 
store
 
+
 
changelog
